## 1. 技术选型

### 前端（小程序）

| 项目 | 选型 | 理由 |
|------|------|------|
| **框架** | 微信小程序原生（WXML + WXSS + TS） | 最佳性能和兼容性，直接调用微信API |
| **语言** | TypeScript | 类型安全，减少运行时错误 |
| **UI 组件库** | TDesign 小程序版 | 腾讯出品，适配微信小程序，组件丰富 |
| **3D 渲染** | Three.js + 小程序 WebGL | 小程序内 3D 模型预览的成熟方案 |
| **状态管理** | MobX-miniprogram | 轻量级，适合小程序 |
| **网络请求** | wx.request 封装 | 统一请求拦截/错误处理/token管理 |

### 后端

| 项目 | 选型 | 理由 |
|------|------|------|
| **运行时** | Node.js (v18+) | 前后端统一 JS/TS 技术栈 |
| **框架** | Koa2 + TypeScript | 轻量级，中间件生态丰富 |
| **数据库** | MySQL 8.0 | 成熟稳定，适合订单/用户等结构化数据 |
| **ORM** | Prisma | 类型安全，迁移管理方便 |
| **缓存** | Redis | 会话管理、生成次数计数、队列状态 |
| **文件存储** | 腾讯云 COS | 照片/模型文件存储，CDN加速 |
| **消息队列** | Redis Queue (BullMQ) | AI 生成任务队列管理 |

### AI 生成

| 项目 | 选型 | 理由 |
|------|------|------|
| **3D 生成引擎** | 待评估（MVP阶段候选方案如下） | — |
| 方案 A | Tripo3D API | 图片转3D模型API，支持Q版风格 |
| 方案 B | Meshy API | 图片生成3D模型，风格化支持 |
| 方案 C | InstantMesh / TripoSR（开源） | 自部署，成本可控 |
| **模型格式** | glTF 2.0 (.glb) | Web端标准3D格式，Three.js 原生支持，体积小 |
| **模型预估大小** | 5-20 MB | glb格式含贴图，需CDN加速 |

> ⚠️ AI 3D 生成引擎是核心技术风险点，建议 MVP 阶段先用 Tripo3D/Meshy 等 SaaS API 快速验证，后续可考虑自建。

### 支付

| 项目 | 选型 | 说明 |
|------|------|------|
| **支付方式** | 微信支付小程序版 | JSAPI 支付 |
| **MVP 阶段** | 模拟支付（¥0.01测试价）| 不接真实支付 |

## 5. 技术风险评估

| 风险 | 严重度 | 应对方案 |
|------|--------|---------|
| AI 3D生成效果不达预期 | 🔴 高 | MVP先对接SaaS API验证，效果不行换方案 |
| 小程序3D渲染性能 | 🟡 中 | Three.js 降级方案：减面/LOD/低端机图片预览 |
| 模型文件过大加载慢 | 🟡 中 | glb格式压缩+CDN加速+骨架屏 |
| 微信支付审核 | 🟡 中 | MVP先用测试价，提前准备资质 |
| AI生成排队等待过长 | 🟢 低 | BullMQ队列管理+优先级+消息通知 |

## 6. 数据库表结构（核心）

```sql
-- 用户表
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  open_id VARCHAR(64) UNIQUE NOT NULL,
  union_id VARCHAR(64),
  nickname VARCHAR(50),
  avatar_url VARCHAR(500),
  phone VARCHAR(20),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 收货地址表
CREATE TABLE addresses (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  name VARCHAR(50) NOT NULL,
  phone VARCHAR(20) NOT NULL,
  province VARCHAR(50) NOT NULL,
  city VARCHAR(50) NOT NULL,
  district VARCHAR(50) NOT NULL,
  detail VARCHAR(200) NOT NULL,
  is_default BOOLEAN DEFAULT FALSE,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 照片表
CREATE TABLE photos (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  original_url VARCHAR(500) NOT NULL,
  cropped_url VARCHAR(500),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 3D模型表
CREATE TABLE models (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  photo_id BIGINT NOT NULL,
  status ENUM('queued','generating','completed','failed') DEFAULT 'queued',
  model_url VARCHAR(500),
  preview_url VARCHAR(500),
  fail_reason VARCHAR(200),
  retry_count INT DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  completed_at DATETIME
);

-- 订单表
CREATE TABLE orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_no VARCHAR(30) UNIQUE NOT NULL,
  user_id BIGINT NOT NULL,
  model_id BIGINT NOT NULL,
  address_id BIGINT NOT NULL,
  status ENUM('pending_payment','cancelled','producing','shipped','completed','refunding','refunded') DEFAULT 'pending_payment',
  figurine_size ENUM('S','M','L') NOT NULL,
  total_price DECIMAL(10,2) NOT NULL,
  discount_price DECIMAL(10,2) DEFAULT 0,
  shipping_fee DECIMAL(10,2) DEFAULT 0,
  pay_price DECIMAL(10,2) NOT NULL,
  package_deal_id VARCHAR(50),
  remark VARCHAR(200),
  paid_at DATETIME,
  shipped_at DATETIME,
  completed_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  expire_at DATETIME
);

-- 订单商品明细表
CREATE TABLE order_items (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  order_id BIGINT NOT NULL,
  item_type ENUM('figurine','model_photo_print','original_photo_print','package_box') NOT NULL,
  item_spec VARCHAR(100),
  price DECIMAL(10,2) NOT NULL
);

-- 每日生成次数表
CREATE TABLE daily_usage (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  usage_date DATE NOT NULL,
  used_count INT DEFAULT 0,
  UNIQUE KEY (user_id, usage_date)
);
```
