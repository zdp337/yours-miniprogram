## 功能概述

"我的"是小程序的个人中心页面，汇集用户个人信息、订单入口、模型管理、地址管理、客服等功能。作为用户的"大本营"，需要清晰的信息架构和便捷的导航。

---

## 详细设计

### 功能点 1：个人中心主页布局

- **描述**: "我的"页面整体布局和信息架构
- **页面结构**（从上到下）:

```
┌────────────────────────────┐
│  头像 + 昵称 + 编辑入口     │  ← 用户信息卡片
│  手机号（已绑定/未绑定）     │
├────────────────────────────┤
│  我的订单                   │  ← 订单快捷入口
│  [全部] [待支付] [制作中]    │
│  [已发货] [已完成]          │
├────────────────────────────┤
│  我的模型                   │  ← 模型管理入口
│  （最近 N 个模型缩略图）     │
├────────────────────────────┤
│  📍 收货地址                │  ← 功能列表
│  📤 分享给好友              │
│  💬 联系客服                │
│  ℹ️  关于我们               │
│  📄 用户协议                │
│  🔒 隐私政策                │
├────────────────────────────┤
│  退出登录                   │  ← 底部
└────────────────────────────┘
```

### 功能点 2：我的订单（入口 + 列表）

- **描述**: 用户查看自己的所有订单，按状态筛选
- **用户操作流程**:
  1. 在"我的"页面点击"我的订单"或对应状态 tab
  2. 进入订单列表页
  3. 顶部 tab：全部 / 待支付 / 制作中 / 已发货 / 已完成
  4. 列表展示每个订单的概要信息
  5. 点击订单 → 进入订单详情页（由策划 B 负责）
- **订单卡片展示信息**:

| 展示项 | 说明 |
|--------|------|
| 模型缩略图 | 手办模型的预览小图 |
| 订单编号 | 格式：YN + 日期 + 序号 |
| 商品摘要 | 如"Q 版手办(M) + 包装盒 + 卡片打印" |
| 总价 | ¥xxx.xx |
| 订单状态 | 待支付 / 制作中 / 已发货 / 已完成 |
| 下单时间 | yyyy-mm-dd hh:mm |

- **逻辑规则**:
  - 订单按下单时间倒序排列（最新在前）
  - 每页加载 10 条，上拉加载更多
  - 空状态展示："还没有订单，去定制你的第一个手办吧~" + 跳转首页按钮
  - **注意**：订单列表页由策划-A负责布局和入口，订单详情页内容由策划-B负责
- **与策划 B 接口**:
  - 点击订单卡片 → 跳转到订单详情页（策划-B）
  - 传递参数：orderId
  - 订单状态枚举需与策划 B 保持一致（由主策统一定义）

### 功能点 3：我的模型

- **描述**: 用户查看自己所有生成过的 3D 手办模型
- **用户操作流程**:
  1. 在"我的"页面点击"我的模型"
  2. 进入模型列表页
  3. 以卡片/网格形式展示所有模型
  4. 每个模型卡片展示：预览图 + 生成时间 + 状态
  5. 点击模型 → 进入 3D 预览页
  6. 预览页可选择"定制我的手办"→ 进入产品定制（策划 B）
- **模型卡片展示信息**:

| 展示项 | 说明 |
|--------|------|
| 模型预览图 | 正面视角渲染图 |
| 生成时间 | yyyy-mm-dd hh:mm |
| 状态 | 生成中 / 已完成 / 生成失败 |
| 操作 | 查看（已完成）/ 等待中（生成中）/ 重试（失败） |

- **逻辑规则**:
  - 模型按生成时间倒序排列
  - 生成中的模型展示 loading 动画
  - 支持删除模型（长按 → 删除确认弹窗）
  - 删除模型不影响已下单的订单
  - 空状态："还没有模型，上传照片生成你的第一个手办吧~"
- **数值/参数**:
  - 模型保存上限：50 个（超出后提示删除旧模型）

### 功能点 4：分享给好友

- **描述**: 从个人中心分享小程序给微信好友
- **分享内容**:
  - 小程序卡片：标题"Yours·凝刻 - 把你变成手办"
  - 配图：品牌宣传图
  - 好友点击 → 进入小程序首页
- **逻辑规则**:
  - 调用 `wx.shareAppMessage` 标准分享
  - 分享后可考虑后续增加邀请奖励（v2）

### 功能点 5：联系客服

- **描述**: 用户联系在线客服
- **实现方式**:
  - 使用微信小程序内置客服（`<button open-type="contact">`）
  - 或跳转到客服电话/企业微信
- **逻辑规则**:
  - 优先使用微信内置客服（无需跳出小程序）
  - 客服工作时间外展示："当前为非工作时间，留言后我们会尽快回复"

### 功能点 6：关于我们 & 协议

- **描述**: 品牌介绍和法律文本
- **页面内容**:
  - **关于我们**: 品牌故事、联系方式、版本号
  - **用户协议**: 标准用户服务协议
  - **隐私政策**: 隐私保护政策（合规必备）
- **逻辑规则**:
  - 纯展示页，富文本渲染
  - 首次注册时需用户勾选同意用户协议和隐私政策
