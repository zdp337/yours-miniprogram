# 模块策划案 — 照片上传与 AI 3D 模型生成

## 基本信息
- **模块名称**: 照片上传与 AI 3D 模型生成
- **负责人**: 策划-A
- **版本**: v1.0
- **最后更新**: 2026-03-12
- **状态**: 初稿

---

## 功能概述

本模块是产品的核心创意环节。用户上传一张全身照，AI 自动生成 Q 版彩色 3D 手办模型。模块涵盖照片上传（含拍照引导）、AI 生成等待流程（页面内等待 + 离开后消息通知）、生成结果处理三个子流程。

---

## 详细设计

### 功能点 1：照片上传入口

- **描述**: 用户从首页发起手办定制流程，进入照片上传页
- **用户操作流程**:
  1. 首页点击"开始定制"（或类似入口）
  2. 进入照片上传页
  3. 选择方式：从相册选取 / 现场拍照
  4. 选取或拍摄照片
  5. 进入照片裁剪/调整页
  6. 确认照片 → 提交 AI 生成
- **逻辑规则**:
  - 仅支持单张照片上传
  - 图片格式：JPG / PNG / HEIC
  - 图片大小限制：≤ 20MB
  - 最低分辨率要求：宽 ≥ 500px，高 ≥ 800px（确保全身清晰）
  - 上传前自动压缩到合理大小（不影响 AI 质量）

### 功能点 2：拍照引导

- **描述**: 在拍照/选照片前，展示引导页帮助用户拍出高质量全身照
- **引导内容**:
  - ✅ **正确示例**：
    - 全身正面站立，双脚可见
    - 光线充足、均匀
    - 背景简洁（纯色墙面最佳）
    - 表情自然
  - ❌ **错误示例**：
    - 只有半身/头部
    - 逆光或光线过暗
    - 背景杂乱
    - 多人合照
    - 遮挡过多（大面积遮脸/身体）
- **展示方式**:
  - 图文引导页，正反面对比
  - 底部"我知道了"按钮 → 进入拍照/选照片
  - 设置"不再提示"选项（记录到本地缓存）
- **逻辑规则**:
  - 首次使用必须展示引导
  - 后续可通过缓存标记跳过
  - 用户也可在选照片后的页面重新查看引导

### 功能点 3：照片裁剪与调整

- **描述**: 用户选取照片后，可进行裁剪和调整
- **用户操作流程**:
  1. 选取照片后进入裁剪页
  2. 展示裁剪框（推荐比例 3:5，竖版全身）
  3. 用户可缩放、移动照片调整裁剪区域
  4. 底部按钮："旋转"、"重选"、"确认"
  5. 确认后预览最终效果
- **逻辑规则**:
  - 裁剪框默认比例 3:5（可自由调整但推荐）
  - 旋转支持 90° 步进
  - 裁剪后的图片分辨率不低于 500×800
  - 裁剪结果本地暂存，提交 AI 时上传

### 功能点 4：AI 3D 模型生成流程

- **描述**: 用户确认照片后，提交给 AI 生成 Q 版 3D 手办模型
- **用户操作流程**:
  1. 确认照片 → 点击"生成我的手办"
  2. 上传照片到服务端
  3. 进入生成等待页
  4. 生成完成 → 自动跳转到 3D 预览页
  5. 生成失败 → 展示失败原因 + 操作选项
- **生成状态枚举**:

| 状态 | 说明 | 界面展示 |
|------|------|---------|
| uploading | 照片上传中 | 上传进度条 |
| queued | 排队中 | "前方还有 N 人，预计等待 X 分钟" |
| generating | 生成中 | 动态动画 + 进度提示 |
| completed | 生成完成 | 自动跳转预览 / 弹窗提示 |
| failed | 生成失败 | 失败原因 + 重试/换照片按钮 |

- **逻辑规则**:
  - 上传时展示进度百分比
  - 排队人数和预计时间由后端返回
  - 生成中每 3 秒轮询一次状态（或使用 WebSocket）
  - 前台等待时展示品牌动画 + 手办制作过程概念动图
  - 用户可随时离开页面

### 功能点 5：后台等待 + 消息通知

- **描述**: 用户离开小程序后，生成完成通过消息通知召回
- **用户操作流程**:
  1. 生成等待页展示"离开后我们会通知你"提示
  2. 弹出订阅消息授权弹窗
  3. 用户授权 → 离开小程序
  4. 生成完成 → 发送微信订阅消息
  5. 用户点击消息 → 打开小程序 → 直接进入 3D 预览页
- **逻辑规则**:
  - 使用微信「一次性订阅消息」（每次需用户主动授权）
  - 消息模板内容：
    - 标题："你的手办生成好了！"
    - 详情："Yours·凝刻手办模型已生成完成，快来查看吧"
    - 跳转：3D 模型预览页（携带 modelId）
  - 如用户拒绝订阅，不影响生成，回到小程序后在"我的模型"中可查看
  - 消息发送失败不影响模型生成

### 功能点 6：生成失败处理

- **描述**: AI 生成失败时的用户体验处理
- **失败原因分类**:

| 失败类型 | 展示文案 | 可用操作 |
|---------|---------|---------|
| 照片质量不足 | "照片清晰度不够，请重新上传一张更清晰的全身照" | 重新选照片 |
| 无法识别人物 | "未能识别到完整的人物，请确保照片中有一个完整站立的人" | 重新选照片 + 查看拍照引导 |
| 服务繁忙 | "当前使用人数较多，请稍后再试" | 重试 |
| 系统异常 | "生成遇到了一些问题，请重试" | 重试 |

- **逻辑规则**:
  - 重试次数限制：同一张照片最多重试 3 次
  - 超过重试次数后建议更换照片
  - 失败不扣费/不消耗次数

### 功能点 7：照片历史管理

- **描述**: 用户可以管理自己上传过的历史照片
- **用户操作流程**:
  1. 在"上传照片"页面可查看历史照片
  2. 点击历史照片可直接使用该照片重新生成
  3. 长按/滑动可删除历史照片
- **逻辑规则**:
  - 保存最近 20 张上传的照片
  - 删除照片不影响已生成的模型
  - 照片存储在云端，跨设备可访问

## 免费次数规则补充

### 生成次数管理
- 每个用户每自然日 2 次免费 AI 生成机会
- 次数在每日 00:00 重置
- 生成失败不消耗次数
- 剩余次数在上传页和生成按钮旁展示："今日剩余 N/2 次"
- 次数用完时：
  - 生成按钮置灰
  - 提示文案："今日免费次数已用完，明天再来试试吧~"
  - 后续可扩展：付费购买额外次数（v2 考虑）

### 模型文件导出功能
- 在 3D 模型预览页增加"导出模型"入口
- 导出格式：由技术评估后确定（优先 glTF / OBJ）
- 导出后可通过微信文件传输 / 下载到手机
- 此功能为 MVP 验证阶段需求，供产品负责人找第三方制作实体手办
