## 功能概述

订单与支付是用户完成商品购买的最终环节。从产品定制页提交后，进入订单确认、微信支付、订单状态跟踪的完整流程。本模块确保用户的支付体验流畅，订单信息清晰透明。

**入口数据**（来自产品定制模块，策划-B 内部）:
- 完整的订单商品数据（含模型信息、定制选项、价格明细）

## 详细设计

### 功能点 1：订单确认页

- **描述**: 用户提交定制方案后，进入订单确认页，确认所有信息无误后支付
- **用户操作流程**:
  1. 从产品定制页点击"提交订单"
  2. 进入订单确认页
  3. 检查/修改收货地址
  4. 查看商品清单和价格明细
  5. 确认无误 → 点击"微信支付 ¥xxx.xx"

- **逻辑规则**:
  - 收货地址默认展示用户默认地址
  - 如用户无地址 → 提示"请先添加收货地址"→ 跳转地址管理页（策划-A）
  - 点击"修改"→ 跳转地址选择页（策划-A）
  - 运费规则：满 ¥xxx 包邮 / 固定运费 ¥xx（待定）
  - 订单备注最多 200 字
  - 订单提交后生成唯一订单编号（格式：YN + yyyyMMdd + 6位序号）
- **异常处理**:
  - 无收货地址：支付按钮置灰，提示添加地址
  - 商品库存不足：提示具体哪个商品缺货
  - 网络异常：提示重试

### 功能点 2：微信支付

- **描述**: 调起微信支付完成付款
- **用户操作流程**:
  1. 点击"微信支付"按钮
  2. 前端请求后端创建预支付订单
  3. 后端返回支付参数
  4. 前端调用 `wx.requestPayment` 调起微信支付
  5. 用户输入密码/指纹/面容完成支付
  6. 支付成功 → 跳转支付成功页
  7. 支付取消/失败 → 留在订单确认页
- **逻辑规则**:
  - 支付超时时间：30 分钟（超时自动取消订单）
  - 支付过程中展示 loading 遮罩，防止重复支付
  - 支付结果以后端回调为准（前端结果仅做界面跳转参考）
  - 后端需实现微信支付回调验证
- **异常处理**:
  - 支付取消：提示"订单已保存，可在'我的订单'中继续支付"
  - 支付失败：提示"支付未成功，请重试"+ 展示重新支付按钮
  - 网络异常：提示"请检查网络后重试"
  - 订单超时未支付：订单自动取消，提示"订单已超时关闭，请重新下单"

### 功能点 4：订单状态流转

- **描述**: 订单从创建到完成的完整生命周期
- **状态枚举与流转**:

```
待支付 ──→ 已取消（超时/主动取消）
  │
  ↓ （支付成功）
已支付/制作中 ──→ 退款中 ──→ 已退款
  │
  ↓ （制作完成发货）
已发货
  │
  ↓ （用户确认收货 / 自动确认）
已完成
```

| 状态 | 状态码 | 说明 | 用户可用操作 |
|------|--------|------|------------|
| 待支付 | pending_payment | 订单已创建，等待支付 | 支付 / 取消订单 |
| 已取消 | cancelled | 超时未支付或主动取消 | 重新下单 |
| 制作中 | producing | 已支付，手办制作中 | 查看进度 / 联系客服 |
| 已发货 | shipped | 手办已发货 | 查看物流 / 确认收货 |
| 已完成 | completed | 用户确认收货 | 评价（预留） |
| 退款中 | refunding | 申请退款处理中 | 查看退款进度 |
| 已退款 | refunded | 退款完成 | — |

- **逻辑规则**:
  - 待支付订单超时时间：30 分钟
  - 已发货后自动确认收货时间：15 天
  - 制作中状态不可申请退款（定制品，已开始制作）
  - 仅"待支付"状态可主动取消
  - 退款审核由后台人工处理

### 功能点 6：物流信息

- **描述**: 展示订单的物流配送信息
- **用户操作流程**:
  1. 在订单详情页点击"查看物流"
  2. 展示物流公司 + 快递单号
  3. 展示物流轨迹时间线
  4. 支持一键复制快递单号
- **逻辑规则**:
  - 物流信息由后端对接快递 100 等第三方物流 API
  - 自动刷新间隔：每次打开页面时拉取最新
  - 未发货状态：展示"商品制作中，发货后可查看物流"
  - 物流异常：展示"物流信息异常，请联系客服"

### 功能点 7：取消订单

- **描述**: 用户主动取消未支付的订单
- **用户操作流程**:
  1. 在订单详情页或订单列表滑动操作
  2. 点击"取消订单"
  3. 弹窗确认："确定取消此订单吗？取消后不可恢复。"
  4. 确认取消 → 订单状态变为"已取消"
- **逻辑规则**:
  - 仅"待支付"状态可取消
  - 取消后订单不可恢复
  - 可以基于同一模型重新下单

## 通用退款政策（MVP 版）

### 退款规则
| 订单状态 | 是否可退 | 退款比例 | 说明 |
|---------|---------|---------|------|
| 待支付 | — | — | 可直接取消订单，无需退款 |
| 已支付（未开始制作） | ✅ 可退 | 全额退款 | 支付后 24 小时内可申请全额退款 |
| 制作中 | ❌ 不可退 | — | 定制品已开始制作，不支持退款 |
| 已发货 | ❌ 不可退 | — | 已发货不支持退款，收货后如有质量问题可联系客服 |
| 已完成 | ⚠️ 仅质量问题 | 全额退款 | 收货后 7 天内如有质量问题可申请退款（需提供照片凭证） |

### 退款流程
1. 用户在订单详情页点击"申请退款"
2. 选择退款原因 + 上传凭证（质量问题时必填）
3. 提交申请 → 后台人工审核（预计 1-3 个工作日）
4. 审核通过 → 原路退回微信支付
5. 审核拒绝 → 展示拒绝原因，可联系客服申诉

### 退款原因选项
- 不想要了（仅已支付未制作状态可选）
- 质量问题（已完成状态可选）
- 商品与描述不符（已完成状态可选）
- 其他（填写原因）

> ⚠️ 以上为 MVP 阶段通用退款政策，正式上线前由法务审核完善。
