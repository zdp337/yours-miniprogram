- **负责人**: 策划-A
- **版本**: v1.0
- **最后更新**: 2026-03-12
- **状态**: 初稿

---

## 功能概述

本模块是用户感知产品价值的关键环节。用户 AI 生成模型后，进入 360° 全角度在线预览，可旋转、缩放查看自己的 Q 版手办模型。确认满意后可进入产品定制环节，不满意可选择重新生成。

---

## 详细设计

### 功能点 1：360° 模型预览

- **描述**: 全角度展示 AI 生成的 Q 版 3D 手办模型
- **用户操作流程**:
  1. AI 生成完成后自动进入预览页（或从"我的模型"点击进入）
  2. 模型居中展示，自动缓慢旋转（吸引用户注意）
  3. 用户可手势操作：
     - 单指滑动 → 旋转模型（水平 360° + 垂直限角度）
     - 双指捏合 → 缩放模型
     - 双击 → 重置视角
  4. 底部操作栏：
     - "重新生成" → 返回照片上传流程
     - "定制我的手办" → 进入产品定制页（策划 B 模块）
- **逻辑规则**:
  - 模型加载使用 3D 渲染引擎（推荐 Three.js / 小程序 WebGL）
  - 自动旋转速度：每秒 15°，用户触摸后停止自动旋转
  - 垂直旋转限制：-30° ~ +60°（防止看到模型底部/顶部穿帮）
  - 缩放范围：0.5x ~ 3x
  - 模型加载时展示 loading 骨架屏
- **数值/参数**:
  - 自动旋转速度：15°/s
  - 垂直旋转限制：-30° ~ +60°
  - 缩放范围：0.5x ~ 3x
  - 模型加载超时：15 秒

### 功能点 2：光影与背景展示

- **描述**: 为模型提供优质的展示环境，增强视觉效果
- **设计规则**:
  - 默认背景：浅灰色渐变（干净、突出模型）
  - 光照方案：三点布光（主光 + 辅光 + 背光），模拟摄影棚效果
  - 模型底部有轻微阴影，增强立体感
  - 后续可扩展：多种背景色/场景可选（v2 考虑）
- **逻辑规则**:
  - 光影参数固定，用户不可调节（v1 简化）
  - 确保不同肤色、服装颜色的模型在该光照下都有良好表现

### 功能点 3：首次查看手势引导

- **描述**: 首次进入预览页时，展示手势操作引导
- **引导内容**:
  - 🔄 "滑动旋转，查看全角度" （配旋转手势动画）
  - 🔍 "双指缩放，查看细节" （配缩放手势动画）
  - 👆 "双击重置视角" （配双击动画）
- **展示方式**:
  - 半透明蒙层 + 手势动画，覆盖在模型上方
  - 每条引导 2 秒，共 3 条，可点击跳过
  - 设置"不再提示"本地缓存标记
- **逻辑规则**:
  - 仅首次查看时展示
  - 引导结束后自动消失
  - 引导过程中用户触摸屏幕可直接跳过

### 功能点 4：模型确认与后续操作

- **描述**: 用户预览后选择下一步操作
- **操作选项**:

| 操作 | 说明 | 跳转 |
|------|------|------|
| 定制我的手办 | 确认使用该模型，进入产品定制 | → 产品定制页（策划-B） |
| 重新生成 | 对模型不满意，重新生成 | → 照片上传页 / 直接用原照片重新生成 |
| 分享模型 | 将模型预览图分享给好友 | → 生成分享图 → 微信分享 |
| 保存到我的模型 | 不定制但保存，稍后再用 | → 保存后返回首页/我的模型 |

- **逻辑规则**:
  - "定制我的手办"传递数据给策划 B 模块：modelId, modelPreviewUrl, originalPhotoUrl
  - "重新生成"：弹窗确认"重新生成不会删除当前模型，确认继续？"
  - 模型自动保存到"我的模型"列表，不需要手动保存
  - "分享模型"生成带小程序码的分享图片

### 功能点 5：分享功能

- **描述**: 将生成的模型分享给微信好友/朋友圈
- **分享方式**:
  - **转发好友**: 小程序卡片，标题"我在 Yours·凝刻生成了一个手办，快来看看！"，配模型预览图
  - **生成海报**: 模型渲染图 + 小程序码 + 品牌 logo，可保存到相册
- **逻辑规则**:
  - 小程序卡片点击 → 进入模型预览页（仅查看，不可编辑/定制）
  - 海报图片尺寸：750×1334（适配朋友圈）
  - 小程序码绑定模型 ID，访客查看时为只读模式
  - 访客如需定制自己的手办 → 引导注册并上传自己的照片

---

### 功能点 6：模型文件导出

- **描述**: 将 AI 生成的 3D 模型文件导出，供用户下载或传输给第三方制作实体手办
- **用户操作流程**:
  1. 在 3D 预览页点击"导出模型"
  2. 弹出导出选项（文件格式由技术评估后确定）
  3. 导出完成 → 可选择"保存到手机"或"通过微信发送"
- **逻辑规则**:
  - 导出格式优先：glTF（推荐）/ OBJ / FBX（由技术评估确定）
  - 导出文件包含：模型几何体 + 贴图纹理 + 材质信息
  - 小程序端可调用 `wx.saveFile` 或生成临时下载链接
  - 此功能为 MVP 验证阶段核心需求
- **异常处理**:
  - 导出失败：提示"导出遇到问题，请重试"
  - 文件过大无法微信传输：提供下载链接方式

## 与其他模块的接口

| 接口描述 | 关联模块 | 数据格式 | 备注 |
|---------|---------|---------|------|
| AI 生成完成 → 进入预览 | 照片上传与 AI 生成（策划-A） | { modelId, modelUrl, status } | 同属策划-A |
| 确认模型 → 进入产品定制 | 产品定制（策划-B） | { modelId, modelPreviewUrl, originalPhotoUrl } | **跨策划接口**，由主策管理 |
| 模型列表查看 | 个人中心（策划-A） | 从我的模型列表进入预览 | 同属策划-A |
| 分享后访客查看 | 自身 | { modelId, viewMode: 'readonly' } | 访客模式无定制按钮 |
