## 功能概述

用户体系是小程序的基础模块，负责用户的注册登录、身份管理和收货地址管理。采用微信生态标准流程，确保用户体验顺畅，同时满足后续订单配送和消息通知的需求。

---

## 详细设计

### 功能点 1：微信授权登录

- **描述**: 用户首次进入小程序时，通过微信授权完成注册/登录
- **用户操作流程**:
  1. 首次打开小程序，展示欢迎页/引导页
  2. 点击"微信登录"按钮
  3. 弹出微信授权弹窗（获取头像、昵称）
  4. 授权成功 → 自动创建账号 → 进入首页
  5. 授权拒绝 → 提示"需要授权才能使用"，可重新触发
- **逻辑规则**:
  - 使用 `wx.getUserProfile` 获取用户信息（微信最新规范）
  - 后端通过 `wx.login` 获取 `code` → 换取 `openid` + `session_key`
  - 首次登录自动创建用户记录
  - 非首次登录自动识别并静默登录
  - 登录态有效期：30 天，过期需重新登录
- **异常处理**:
  - 网络异常：提示"网络不给力，请重试"
  - 授权拒绝：引导用户到设置页手动开启授权
  - 服务端异常：提示"服务繁忙，请稍后再试"

### 功能点 2：手机号绑定

- **描述**: 用户绑定手机号，用于收货联系和订单通知
- **用户操作流程**:
  1. 登录后首次进入需要手机号的流程时（如下单），提示绑定
  2. 点击"绑定手机号"按钮
  3. 弹出微信手机号授权（使用 `getPhoneNumber` 按钮组件）
  4. 授权成功 → 手机号绑定完成
  5. 也支持手动输入手机号 + 短信验证码方式绑定
- **逻辑规则**:
  - 优先推荐微信手机号一键授权（体验更好）
  - 备选方案：手动输入 + 短信验证码（60 秒冷却）
  - 手机号可更换，需重新验证
  - 一个手机号只能绑定一个账号
- **异常处理**:
  - 授权拒绝：引导手动输入方式
  - 短信发送失败：提示重试，1 分钟内不可重复发送
  - 手机号已被其他账号绑定：提示"该手机号已绑定其他账号"

### 功能点 3：个人信息管理

- **描述**: 用户可查看和编辑个人基本信息
- **用户操作流程**:
  1. 在"我的"页面点击头像区域或"编辑资料"
  2. 进入个人信息编辑页
  3. 可修改：头像（从相册/拍照）、昵称
  4. 可查看（不可改）：手机号（需到专门入口更换）、微信号
  5. 点击"保存"提交修改
- **逻辑规则**:
  - 昵称长度限制：1-20 个字符
  - 昵称敏感词过滤
  - 头像支持裁剪为正方形
  - 修改后立即生效，全局同步
- **异常处理**:
  - 昵称包含敏感词：提示"昵称包含违规内容，请修改"
  - 头像上传失败：提示重试
  - 保存失败：提示网络异常

### 功能点 4：收货地址管理

- **描述**: 用户管理收货地址，用于手办配送
- **用户操作流程**:
  1. 在"我的"页面 → 收货地址
  2. 查看地址列表（默认地址置顶，标记"默认"标签）
  3. 点击"新增地址"→ 填写收货人/手机号/地区/详细地址
  4. 支持调用微信地址簿一键导入
  5. 可编辑/删除已有地址
  6. 可设置/切换默认地址
- **逻辑规则**:
  - 最多保存 20 个地址
  - 必填项：收货人、手机号、省市区、详细地址
  - 手机号格式校验（11 位数字）
  - 默认地址有且只有一个
  - 删除默认地址后，自动将最新地址设为默认
  - 下单时如无地址，引导用户新增
- **数值/参数**:
  - 地址上限：20 条
  - 收货人姓名：1-20 字符
  - 详细地址：5-200 字符
- **异常处理**:
  - 地址数量达上限：提示"最多保存 20 个地址，请删除后再添加"
  - 必填项未填：对应输入框标红 + 提示
  - 保存失败：提示网络异常
